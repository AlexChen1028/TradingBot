## 一、價格走勢與關鍵支撐／壓力

- **熊市中罕見的強勢收斂**：BTC 被擠壓在「短期持有者成本線（約 78,500 USD）」與「200 日均線（牛熊分界線，約 82,000 USD）」之間，這在過去大熊市中極罕見。〔80600 快沒時間了〕
- **死守關鍵支撐位**：價格回踩時必須密切關注 **78,500 ~ 79,300 USD** 區間。跌破此防線恐觸發短期持有者虧損出逃的拋壓。〔跌穿 8 萬怎麼辦〕
- **利好帶動強力挑戰均線**：CLARITY Act 投票通過第一階段後，BTC 守住 79,300 並一週內三度挑戰 200 日均線 (~82,000)，展現極強多頭走勢。〔CLARITY Act 終於通過第一階〕

## 二、合約市場與資金費率變化（軋空燃料）

- **多頭偷跑進場的隱憂**：盤整末端曾出現大量多頭進場，幣安資金費率一度達 **0.01**。此時若貿然突破關鍵位，需小心因高槓桿多單過多而觸發急跌洗盤。〔80600 快沒時間了〕
- **多頭未退，缺乏軋空燃料**：價格向下回踩時資金費率仍維持正值 → 多頭未離場 → 暫無足夠空單堆積 → 缺乏向上軋空燃料。〔跌穿 8 萬怎麼辦〕
- **空頭重新堆積，軋空動能增加**：消息面上漲觸及 82,000 USD 附近後，資金費率迅速轉負 → 大量空頭在 200 日均線附近佈局對賭 → 為後續突破壓力位、向上軋空提供燃料。〔CLARITY Act 終於通過第一階〕

## 三、鏈上數據、現貨 ETF 與法案影響

- **法案投票前夕的避險拋售潮**：參議院銀行委員會投票前夕，出現近兩週最大鏈上大鯨魚賣出量，以及自一月底以來最大現貨 ETF 資金淨流出（高達 **630M**）。〔跌穿 8 萬怎麼辦〕
- **避險情緒消散，資金回流**：法案順利通過第一關後，ETF 恢復 **131M 淨流入**，大鯨魚也出現近兩週最大規模買入。〔CLARITY Act 終於通過第一階〕
- **美股與加密的流動性蹺蹺板**：美股漲勢過熱並開始回調時，流動性反而會釋放回加密市場 → 美股回調可能是下一輪大牛市啟動的催化劑，而非末日。〔CLARITY Act 終於通過第一階〕
- **機構吸籌與週期改變**：MicroStrategy + 現貨 ETF = 華爾街「新鯨魚」搶奪籌碼，可能已徹底打破比特幣過往「四年一輪、漲幅遞減」的傳統牛熊規律，轉變為類似美股的「更大週期牛市」。〔80600 快沒時間了〕

## 四、宏觀與機構判斷如何塑造交易決策邏輯

- **避免受消息面干擾搶快做空**：價格被夾在 200 日均線（上方壓力）與短期持有者成本線（下方支撐）的極小區間時，不要在下方支撐未跌破前「搶快做空」 — 這等同於空在支撐位上，容易因主力誘空而成為嘎空燃料。〔80600 快沒時間了〕
- **監測資金費率，尋找嘎空動能**：當價格挑戰關鍵壓力位（如 200 日均線）、大量空頭湧入導致資費轉負時 → 視為「增加了繼續往上嘎空的力量」，因為市場上有充足流動性供主力獵殺。〔CLARITY Act 終於通過第一階〕
- **左側抄底配合右側重倉**：對機構主導的市場，拒絕被「看跌到 4 萬」的焦慮影響。前期回踩 6 萬時透過左側交易極限買入 60% 倉位，剩餘 40% 等待「右側」確認（明確站上均線突破整理）再進場。雖然成本高 3-4 成，但勝在「確定性高」。〔80600 快沒時間了〕

## 對 `crypto-bot` 的可實作映射

### A. 缺失但關鍵的特徵

| 特徵 | 對應 KOL 觀點 | 實作難度 | 數據來源 |
|---|---|---|---|
| **200 日均線比率 (ma200_ratio)** | 牛熊分界線、價格區間判讀 | 低 | OHLCV |
| **短期持有者成本線距離** | 下方關鍵支撐 | 中 | 鏈上數據（Glassnode／CryptoQuant） |
| **資金費率符號翻轉旗標 (fr_flip_negative)** | 嘎空動能訊號 | 低 | 已有 funding_rate_raw |
| **價格接近壓力位 × 資費為負 = squeeze fuel** | 軋空組合訊號 | 低 | 派生特徵 |
| **ETF 淨流量** | 機構資金動向 | 中 | Farside Investors／SoSoValue |
| **大鯨魚淨流量** | 拋售/吸籌動向 | 高 | 鏈上數據 |

### B. 已有但可強化的部分

- `data.py` 已有 `fr_norm / fr_z / fr_ma / fr_cumsum` — KOL 一直強調的核心訊號，已具備
- `data.py` 已有 `fr_trend_corr` — 對應 KOL「資費跟著趨勢 = 動能型；資費背離 = 反轉型」的概念
- `main.py` `detect_regime()` 用 ATR ratio 分 trending/ranging — 對應 KOL 的「收斂盤整 vs 突破」框架
- `main.py` `MIN_HOLD_HOURS=6` + 多時框確認 — 對應 KOL「耐心」「右側交易」哲學

### C. 訊號條件落地（可程式化規則）

下面這些是直接可以變成 `if-else` 的條件：

```python
# 嘎空燃料偵測（向上）
SQUEEZE_FUEL_UP = (
    close_price > ema200 * 0.98 and          # 接近 200 日均線
    funding_rate < -0.0001 and               # 資費為負
    rsi < 70                                 # 還沒過熱
)

# 假突破風險（向下洗盤）
FAKE_BREAKOUT_RISK = (
    funding_rate > 0.0005 and                # 多頭過熱
    abs(close - resistance) / close < 0.005  # 緊貼壓力
)

# 短期持有者拋壓觸發
STH_CAPITULATION = (
    close_price < short_term_holder_cost_basis and  # 跌破成本線
    funding_rate > 0                                 # 多頭尚未投降
)

# 右側交易確認（高確定性進場）
RIGHT_SIDE_LONG_ENTRY = (
    close_price > ema200 and                 # 站上牛熊分界
    consecutive_closes_above_ema200 >= 3 and # 連續站穩
    regime == 'trending'                     # 趨勢市
)
```

### D. 風險警示（KOL 強調但 bot 已部分覆蓋）

- 不要在支撐位下方搶空 → bot 的 THRESHOLD=0.50 已避免在中性區進場，但可加入「靠近強支撐時不開空」的條件
- 多頭高度集中時的洗盤風險 → 可在 `open_position` 加入「資費 > 0.0005 時暫不重倉做多」的緩衝
- 法案投票/重大事件日的避險拋售 → 可加入 macro event blackout window，事件前 24h 不開新倉
